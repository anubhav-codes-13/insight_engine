Insight Engine
Backend

A Python ETL pipeline that processes customer support case data and serves pre-computed dashboard metrics via a FastAPI REST endpoint.

FastAPI Pandas ETL Uvicorn 1,000 Cases 12 JSON Outputs CORS Enabled

Project Structure

File Layout

All project files live in a single flat directory alongside the virtual environment.

Insight_Engine_Backend/ ├── case_level_data.csv # Source data (1000 support cases) ├── etl_pipeline.py # ETL logic — filtering, metrics, aggregation ├── api.py # FastAPI server — serves pre-computed JSON ├── output/ # 12 pre-computed dashboard JSON files │ ├── dashboard_1d_all.json │ ├── dashboard_7d_chat.json │ └── ... (3 time windows × 4 channels) ├── venv/ # Python virtual environment └── README.md

Quick Start

Running the Backend

Two commands to get fully operational — run the ETL first, then start the server.

⚙️
Step 1 — Run the ETL pipeline
Generates all 12 pre-computed JSON files in output/
bash
venv\Scripts\python.exe etl_pipeline.py
🚀
Step 2 — Start the API server
Starts Uvicorn with hot-reload on port 8000
bash
venv\Scripts\python.exe api.py
● Live at http://localhost:8000 Docs at http://localhost:8000/docs

ETL Pipeline

etl_pipeline.py — Function Reference

11 functions that orchestrate data loading, filtering, metric computation, and JSON output generation.

load_data
Loads the CSV and parses the date column.
Input Parameters
ParameterTypeDefaultDescription
filepath str "case_level_data.csv" Path to the CSV file
Output
pd.DataFrame — raw dataframe with date column as datetime64
filter_data
Splits the dataframe into a current and previous time window, optionally filtered by channel.
Input Parameters
ParameterTypeDefaultDescription
dfpd.DataFrame—Full raw dataframe
time_window_daysint—Window size in days (e.g. 1, 7, 30)
channelstr—"All" to skip filter, or "Chat" / "Call" / "Email" / "Phone"
reference_datedatetimeNoneAnchor date; defaults to max date in df
Output — (current_df, previous_df)
DataFrameDate Range
current_df[reference_date − N days, reference_date]
previous_df[reference_date − 2N days, reference_date − N days)
calculate_kpis
Computes 5 core KPIs for the current period and their % change vs the previous period.
Input Parameters
ParameterTypeDescription
current_dfpd.DataFrameCurrent window data
previous_dfpd.DataFramePrevious window data
Output — dict
json
{
  "total_requests":  {"value": 250,  "trend": 8.2},
  "resolution_rate": {"value": 43.6, "trend": -3.1},
  "aht":             {"value": 17.4, "trend": 0.0},
  "csat":            {"value": 55.8, "trend": -5.0},
  "drop_off":        {"value": 1.1,  "trend": null}
}
⚠️ trend is null when previous window is empty (no prior baseline). All values rounded to 1 decimal place.
generate_chart_data
Produces time-bucketed trend data and L1-level issue cluster data for charts.
Input Parameters
ParameterTypeDescription
current_dfpd.DataFrameCurrent window data
previous_dfpd.DataFramePrevious window data
time_window_daysintDrives bucketing — hourly if 1, daily otherwise
Output — dict
json
{
  "trends": [
    {"date": "2026-03-12", "volume": 35, "resolution": 48.6, "escalation": 22.9}
  ],
  "clusters": [
    {"topic": "Display & Screen", "vol": 41, "prev_vol": 27, "chg": 51.9, "trend_data": [4,8,7,6,8,8]}
  ]
}
⚠️ trends.date format: "YYYY-MM-DD" for daily, "YYYY-MM-DD HH:00" for hourly. clusters.chg is null when prev_vol is 0.

Lightweight FastAPI server that serves pre-computed JSON. No computation happens at request time.

SettingValue
FrameworkFastAPI
ServerUvicorn
Host0.0.0.0:8000
CORS OriginsAll origins allowed ( * )
CORS Methods*
CORS Headers*
GET /api/dashboard

Serves a pre-computed dashboard payload JSON file based on query parameters.

GET http://localhost:8000/api/dashboard
Query Parameters
ParameterTypeDefaultValid Values
time_window str "7d" "24h"  |  "7d"  |  "30d"
channel str "all" "all"  |  "chat"  |  "call"  |  "email"
Example Requests
http
GET /api/dashboard
GET /api/dashboard?time_window=30d&channel=chat
GET /api/dashboard?time_window=24h&channel=all
200 OK Returns the full dashboard payload JSON
response schema
{
  "filters":         { time, channel, start_date, end_date },
  "kpis":            { total_requests, resolution_rate, aht, csat, drop_off },
  "charts":          { "trends": [...], "clusters": [...] },
  "ai_insight":      "...",
  "monitoring":      { trending, performing, category_metrics },
  "issues_insights": [...]
}
404 Not Found Invalid parameter combination
error response
{
  "detail": "No dashboard file found for time_window='90d' and channel='chat'.
            Valid time windows: 24h, 7d, 30d.
            Valid channels: all, chat, call, email."
}

Maintenance

Regenerating the Output Files

Run etl_pipeline.py any time the source data changes. The API serves pre-computed files, so the server does not need to restart.

bash
venv\Scripts\python.exe etl_pipeline.py
The pipeline will
1
Load case_level_data.csv
2
Auto-detect the max date as the reference anchor
3
Compute all metrics and write 12 JSON files to output/
